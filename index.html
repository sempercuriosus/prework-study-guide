HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML is the structure, on which, style and function are added.
  • Semantic HTML is the best practice, when possible, some tags are not semantic friendly but are the best way to solve the framing problem
  • Tags are not the same thing as elements
    • tags are just the braces '<' '>' and name
    • elements are the tag and the content

CSS

  • A margin indicates how much space we want around the OUTside of an element.
  • A padding indicates how much space we want around the content INside an element.
  • External CSS is the most popular and easy to maintain CSS; Internal is next and can only apply to the current webpage; and Inline can only apply to that element and is good for quick changes/ test but is hard to maintain
  • CSS Boc Model (outside to inside) Margin, Border, Padding, Content
  • Anatomy of CSS Rule Declarations: Selector {Property: Value, Property: Value }
    • The Declaration is a Propery: Value pairing and the Declaration Block is one or more Declarations
    • Selectors can stack and is encouraged for D.R.Y code
    • Splats (*) can be used as a wildcar Selector
  • rel attribute states the type of relationship that the document and linked resources have
  • href is location to the resource
